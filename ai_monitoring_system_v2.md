### 4. 自动化流程

```
每日监测 (4次)
├─ 早间 09:00 → 夜间12小时动态
├─ 下午 14:00 → 午间动态
├─ 晚间 20:00 → 下午动态
└─ 深夜 23:00 → 晚间动态

每周汇总
└─ 周五 18:00 → 生成本周报告

每月优化
└─ 月末 → 自动分析名单活跃度，提议调整
```

### 5. 自动调整名单规则

**自动添加触发条件：**
- 被现有追踪对象≥3次提及的新人物
- 在AI领域重大新闻中频繁出现的人物
- 用户手动指令添加

**自动暂停触发条件：**
- 连续14天无动态
- 内容质量持续下降(低价值占比>80%)
- 转行至非AI领域

**自动恢复触发条件：**
- 暂停期间有新重大动态(被其他P0提及)

**自动移除触发条件：**
- 连续30天无动态且非重要人物
- 账号注销/停用
- 用户手动指令移除

### 6. 监测执行步骤

**每次监测流程：**
1. 轮询所有活跃人员的各平台账号
2. 抓取新动态(基于上次检查时间)
3. AI分析内容(分类/重要性/情绪)
4. 写入Bitable表2
5. 检查提及的新人物(是否加入候选)
6. 更新人员档案表1的活跃度数据

**周报生成流程：**
1. 汇总本周所有动态(表2筛选)
2. AI分析趋势和重点事件
3. 生成结构化报告
4. 写入表3并发送通知
