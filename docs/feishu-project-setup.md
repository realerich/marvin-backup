# 飞书项目配置指南

## 推荐配置

**项目类型：** 敏捷开发 / 通用项目管理
**项目名称：** Marvin协作空间

---

## 工作流设置（看板列）

```
📥 需求池 (Backlog)
   └─ 所有新想法、待评估任务
   
📋 待办 (To Do)
   └─ 已确认，等待开始
   
🔄 进行中 (In Progress)
   └─ 正在执行
   
👀 审核中 (Review)
   └─ 已完成，等待验收
   
✅ 已完成 (Done)
   └─ 验收通过，归档
```

---

## 字段配置

**基础字段：**
| 字段 | 类型 | 说明 |
|------|------|------|
| 任务名称 | 文本 | 简洁描述 |
| 描述 | 富文本 | 详细需求 |
| 优先级 | 单选 | P0紧急 / P1重要 / P2普通 |
| 负责人 | 人员 | 执行者 |
| 截止日期 | 日期 | 完成期限 |
| 标签 | 多选 | 分类标记 |

**自定义字段：**
| 字段 | 类型 | 选项 |
|------|------|------|
| 任务类型 | 单选 | 开发 / 数据分析 / 调研 / 运维 / 其他 |
| 预估工时 | 数字 | 小时数 |
| 关联系统 | 多选 | RDS / GitHub / 定时任务 / 飞书 |
| 数据来源 | 文本 | 如：股票API、邮件等 |

---

## 标签体系

**按类型：**
- `🔧 基础设施` - 服务器、数据库、配置
- `📊 数据分析` - 监测、报告、统计
- `🤖 AI/ML` - 智能分析、自动化
- `🔌 集成` - API对接、第三方服务
- `🐛 Bug修复` - 问题处理

**按优先级：**
- `🔴 P0` - 阻塞性问题，当天处理
- `🟡 P1` - 重要任务，本周完成
- `🟢 P2` - 普通任务，按需处理

## 协作流程

**创建任务：**
1. 有新想法 → 创建任务到「需求池」
2. 评估优先级 → 标记P0/P1/P2
3. 确认可执行 → 移到「待办」

**执行任务：**
1. 开始工作 → 移到「进行中」
2. 完成任务 → 移到「审核中」
3. 我汇报结果 → 飞书通知你
4. 验收通过 → 移到「已完成」

**日常同步：**
- 每日晨报：自动汇报昨日完成、今日计划
- 任务变更：实时飞书通知
- 阻塞问题：立即升级到P0

## 创建步骤

**Step 1: 创建项目**
```
飞书 → 工作台 → 项目 → 新建项目
→ 选择「敏捷开发」模板
→ 项目名称：Marvin协作空间
```

**Step 2: 配置工作流**
```
项目设置 → 工作流
→ 添加列：需求池、待办、进行中、审核中、已完成
→ 设置流转规则（如：进行中→审核中需要填写完成说明）
```

**Step 3: 添加字段**
```
项目设置 → 字段
→ 添加上述自定义字段
→ 设置默认值和选项
```

**Step 4: 邀请成员**
```
项目设置 → 成员
→ 添加你为项目管理员
→ 添加我为项目成员（如果飞书支持）
```

**Step 5: 创建首个任务**
```
使用「飞书项目配置」任务测试流程
```

## 快捷操作

**任务ID规范：**
- 格式：`[类型]-[序号]`，如 `FEAT-001`, `BUG-002`
- 便于引用和检索

**飞书机器人集成（可选）：**
- 任务状态变更 → 自动发飞书消息
- 截止日期提醒 → 提前1天通知
- 每日晨报 → 自动汇总任务进展
